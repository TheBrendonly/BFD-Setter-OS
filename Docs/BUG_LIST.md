## 🟠 Medium

- [ ] **F9-1 (found in Session 7 TEST, 2026-06-28) — Retell lock leaks the tile display-name rename + misreports it as saved.** With a voice setter **Retell-locked**, the inline tile name editor still lets you rename it and reports success. Root cause: `InlineSetterNameEditor.commitEdit` (`frontend/src/components/setters/InlineSetterNameEditor.tsx`) writes `clients.setter_display_names` **unconditionally first**, *then* calls the guarded `retell-proxy set-agent-name`. The server **correctly refuses** the Retell write (HTTP 423 `setter_retell_locked` → live Retell `agent_name` **and** `voice_setters.name` stay unchanged — core lock safety holds, no Retell corruption), but the display-name map already persisted, so the locked tile shows the new name and the toast is a soft *"Setter name saved (Retell push warning)"* instead of a clean "Retell-locked" refusal. **Repro (live-confirmed):** lock slot 4 → click tile name heading → "Gary - Property Coach LOCKTEST" → Enter → `setter_display_names.voice-4` changed; `voice_setters.name` + Retell `agent_name` unchanged. **Expected (F9 contract):** the rename is refused with a clear "Retell-locked — unlock to rename" message and **no** display-name write. **Fix direction:** the tile already has the lock state (`isVoiceSlotLocked(slotId)` / `lockMap` at `PromptManagement.tsx:8168`) — pass it into `InlineSetterNameEditor` and gate `commitEdit` (block + error toast, skip the `setter_display_names` write); and/or reorder so the guarded `set-agent-name` runs first and the display-name write only persists on success, and surface the 423 body as an **error** (not a warning). Severity Medium (UX + tile/Retell name mismatch; no data-safety breach). Note: `setter_display_names.voice-4` was reverted to "Gary - Property Coach" during the test to keep the live tile correct.

- [ ] **VM-1 (found in Session 7 TEST, 2026-06-28) — `set-voicemail` client-wide push reports "partial" and never lands (sends fields Retell no longer accepts).** Clicking **Save & push** on the client Voicemail card shows *"Push partial: voicemail_set"* — that is the `success:false` branch (`ClientVoicemailCard.tsx:150`), i.e. ≥1 **unlocked** agent's PATCH returned `ok:false`. Root cause (read-only evidence): `retell-proxy` `set-voicemail` (`index.ts:1573-1586`) PATCHes `update-agent` with **`enable_voicemail_detection` + `voicemail_detection_timeout_ms`**, but the live Retell agent schema **no longer has those fields** (`has("enable_voicemail_detection")` → `false`; only `voicemail_option` exists) → Retell rejects the PATCH; nothing lands (all 6 agents' `last_modification_timestamp` unchanged, `enable_voicemail_detection` null). It also does a **raw `update-agent` PATCH without `ensureEditableAgentDraft`** (every other write handler draft-first). Impact: the existing `voicemail_option:{hangup}` is already on the agents (set earlier), so *hangup* still works, but **no voicemail change can be pushed** — a `static`/`prompt` voicemail message would never apply, and every push shows a scary "partial". **Pre-existing (not F9)** — F9 only added the lock-skip loop. **Fix:** drop the two deprecated fields from `patchBody` (map the intended behavior onto the current `voicemail_option` schema), and route through `ensureEditableAgentDraft` like the other handlers. Confirm the exact Retell error from the `results[].error` in the `set-voicemail` response. Severity Medium.

- [ ] **API-DEPR-1 (raised by Brendan 2026-06-28, Session 7) — Retell + GHL API deprecation / version review (CODE; dedicated session).** Brendan has received Retell's **legacy-list-endpoints deprecation notice twice** (https://docs.retellai.com/deprecation-notice/2026/06-15_legacy_list_endpoints). Read-only scan confirms a **mixed/partial migration state**: `make-retell-outbound-call` already uses `/v2/list-phone-numbers`, but **`retell-proxy` still calls the bare legacy list endpoints** — `list-agents` (1270-71), `list-calls` (1415-17), `list-phone-numbers` (1387-88), `/list-retell-llms` (1297), `list-knowledge-bases` (1337-38), `list-voices` (919/1152/1431-32) — and `verify-credentials` uses `/list-agents`, `elevenlabs-manage-agent` uses `list-voices`. Verify read-only against the live APIs, deploy the touched edge fns, re-confirm no deprecation notice fires. Severity Medium (deprecated endpoints get removed → future hard breakage; actively firing now).
