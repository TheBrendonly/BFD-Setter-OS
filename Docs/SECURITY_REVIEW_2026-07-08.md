## Architecture facts that drive severity (re-confirmed 2026-07-08)
- Browser talks to the platform DB with the anon key; **RLS is the only tenant boundary** for direct reads. Live: `anon` AND `authenticated` hold column-level SELECT/INSERT/UPDATE on the `clients` secret columns (verified via `information_schema.role_column_grants`), so RLS policy quals are the entire defense.
- **Every** edge function deploys `verify_jwt = false`; in-function authz is mandatory. The canonical guards are `resolveClientAccess` / `authorizeClientRequest` / `assert-client-access.ts`.
- Agency `FOR ALL` RLS also matches a **client-role** user (a client belongs to its own agency), so any cost/pricing/secret/margin table MUST additionally gate `get_user_role(auth.uid()) = 'agency'`, and any client-writable table MUST scope to the user's own client via `get_user_client_id(auth.uid())`.
- **Live enabling-state that gates real-world exploitability (verified this pass):** `retell_webhook_secret` is NULL for both clients (so the 3 Retell webhooks are effectively unauthenticated); `ghl_webhook_secret` IS set for both (so the GHL webhooks are signature-enforced TODAY and a forged POST 403s); `missed_call_textback_enabled` = 0 clients; **0 client-role users exist**; the 2 live clients share ONE agency (`fb6b57a3…`).
