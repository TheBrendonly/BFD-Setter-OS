## What B-6 actually was

Brendan reported the inbound "INBOUND CALLS — USE THIS SETTER?" toggle (Inbound BFD Agent, slot 8)
"doesn't hold" and the setter reads **"Not Active"** no matter how many saves. **Read-only investigation
this session proved the persistence path already works on the current build** — the whole inbound chain
is correctly bound right now:

| Surface | Live state (verified read-only) |
|---|---|
| `voice_setters` slot 8 "Inbound BFD Agent" | `is_inbound=true`, `retell_agent_id=agent_b2f6495…`, `legacy_slot=8` (no duplicate slots) |
| `clients.retell_inbound_agent_id` | `agent_b2f6495…` |
| Retell `+61481614530` inbound binding | `inbound_agents=[{agent_id: agent_b2f6495…, agent_version: latest_published, weight:1}]` |

So Brendan's original failure was almost certainly testing mid-Railway-deploy on the OLD build (the
leading hypothesis in B-6). The real, still-present bug was a **split-brain in the list-view status
badges**: the source of truth is `voice_setters.is_inbound`, but the list badges read the wrong table —
the "Active/Not Active" badge read `prompts.is_active` (→ "Not Active") and the "Inbound" badge read
`prompts.directions` (→ `[]`, never showed). Both `prompts.*` are false/empty **by design** for the
inbound agent: they only flip on Deploy/"Save Setter Prompt", and the inbound agent must **never** be
deployed via the UI (report-only-prompt rule — deploying overwrites the live neutral greeting on
`agent_b2f6495`). The badge logic was simply keyed off the wrong column.

## What shipped

- **`pages/PromptManagement.tsx`** — new uncached state (`inboundSlotMap` keyed by `legacy_slot`,
  `clientInboundAgentId`, `inboundMapRefresh`) + a list-only fetch effect that reads
  `voice_setters (legacy_slot, is_inbound, retell_agent_id)` + `clients_public.retell_inbound_agent_id`
  (kept OUT of the prompts SWR cache so the badge never shows stale inbound state). Both list badges
  repointed to this SoT: the left "Inbound" badge now reads `is_inbound`; the right status badge, for an
  inbound-flagged slot, shows green **"Bound"** when `retell_agent_id === clientInboundAgentId` else amber
  **"Inbound · rebind"** (DB-only, no Retell call) — and never the misleading red "Not Active". Non-inbound
  slots fall through to the unchanged `prompts.is_active` logic. Map refreshes after a successful toggle.
- **`hooks/useSetInboundSetter.ts`** — the two `is_inbound` UPDATEs (set-true, toggle-OFF) now append
  `.select('id')` and, on a 0-row write (silently RLS-filtered / stale id), `toast.error(...)` + return
  false so the toggle reverts and a failed write is never silent. The "clear others" update is left
  untouched (it legitimately affects 0 rows when no other setter is inbound).
- **`components/prompt-doc/DirectionsToggle.tsx`** — new optional `disabled` prop wired to the
  `ToggleGroup` (Radix forwards it to the item); held true via the hook's `binding` flag while a write is
  in flight, so a second toggle can't race the first. Stale "persisted to prompts.directions" comment
  corrected to the `voice_setters.is_inbound` SoT.

Files: `pages/PromptManagement.tsx`, `hooks/useSetInboundSetter.ts`,
`components/prompt-doc/DirectionsToggle.tsx` (mod). `components/StatusTag.tsx` reused, unchanged.
